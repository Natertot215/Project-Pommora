### v0.1 ‚Äî Nexus Foundation: Picker, Bookmark, Sidebar Tree

> Brainstormed 2026-05-15. After approval, move this file to `// Planning// v0.1-nexus-foundation-design.md` per Studio convention.

#### Context

Pommora v0.0 shipped a barebones three-pane shell with hardcoded placeholder rows ([ContentView.swift:26‚Äì69](Pommora/Pommora/ContentView.swift#L26-L69)). v0.1's first capability cluster is reading a real nexus folder from disk and surfacing its structure in the sidebar.

This design covers the foundation layer of v0.1: nexus picker, sandbox + security-scoped bookmark persistence, hidden `.pommora/` config folder initialization, and folder-tree-mirroring sidebar rendering with extensions stripped. It does **not** cover opening files in tabs, the file watcher, schema parsing, or `.json` Item windows ‚Äî those land in v0.1's later increments and v0.2+ per [Framework.md](.claude/Framework.md).

The dual-location pattern Nathan asked about (development source in The Studio, real nexus on his disk) is achieved automatically by standard macOS app architecture ‚Äî the Xcode project location and the runtime data location are independent. The nexus is wherever Nathan picks it; the app remembers via bookmark.

#### Decisions Made During Brainstorming

| Decision | Choice | Rationale |
|---|---|---|
| App Sandbox | Enabled from v0.1 | Future-proofs MAS distribution; clean architecture from day one |
| `pommora.db` location | `~/Library/Application Support/com.nathantaichman.Pommora/nexuses/<nexus-id>/pommora.db` | Resolves the iCloud-corruption known gap; GRDB.swift official recommendation; SQLite official guidance against network filesystems; Apple Foundation canonical pattern. Bundle ID is the current Xcode default; can be re-namespaced (e.g., `com.pommora.Pommora`) before MAS distribution if desired |
| Per-nexus DB subdir keyed by ID | Yes, ULID stored in `.pommora/nexus.json` | Survives nexus folder renames/moves |
| First-launch UX | `NSOpenPanel` opens immediately | Minimum friction for personal-use dev iteration |
| Sidebar scope | Folders + `.md` + `.json`; skip leading-dot entries and other file types | Honors PRD "curated app navigation" principle |
| Filename rendering | Strip `.md` / `.json` extension on display | Locked decision: filename = title |
| Code organization | Single app target, folder organization (`Nexus/`, `Sidebar/` folders) | YAGNI; defer Swift Package extraction until a second consumer exists |
| State storage | Dedicated `state.json` from v0.1 ‚Äî app-level state at `~/Library/Application Support/com.nathantaichman.Pommora/state.json`; nexus identity at `.pommora/nexus.json` inside nexus (overrides Navigation-Bar.md note) | Architecturally clean; no UserDefaults dependency; clear separation of machine-specific state (App Support) from nexus-portable identity (in nexus) |
| `.pommora/` initialization on empty folder | Silent | Zero friction for the explicit "create new nexus" flow |
| Initialization on non-empty folder without `.pommora/` | Confirm dialog | Safety: prevents accidentally polluting `~/Documents/` |

##### Component responsibilities

**`Nexus`** ‚Äî value type. `id: ULID`, `rootURL: URL`. Equatable for SwiftUI diffing.

**`NexusManager`** ‚Äî `@Observable` class. Single source of truth for the active nexus.
- `currentNexus: Nexus?`
- `pickNexus() async` ‚Äî presents `NSOpenPanel`, handles cancel, runs the init flow on the picked folder, saves bookmark + state.
- `loadOnLaunch()` ‚Äî called from `App.task`; resolves saved bookmark, refreshes if stale, sets `currentNexus`. If no bookmark, calls `pickNexus()` immediately.
- `resetBookmark()` ‚Äî DEBUG-only; deletes `~/Library/Application Support/com.nathantaichman.Pommora/state.json`, triggers picker on next launch.
- Owns `stopAccessingSecurityScopedResource()` cleanup in `deinit`.

**`NexusStore`** ‚Äî pure functions for path resolution.
- `applicationSupportDir() throws -> URL` ‚Äî wraps `FileManager.default.url(for: .applicationSupportDirectory, ...)`.
- `pommoraAppDir() throws -> URL` ‚Äî `appSupport/com.nathantaichman.Pommora/`, created lazily.
- `nexusDataDir(nexusID:) throws -> URL` ‚Äî `pommoraAppDir/nexuses/<id>/`, created lazily, marked `isExcludedFromBackupKey = true`.
- `databaseURL(nexusID:) -> URL` ‚Äî `nexusDataDir/pommora.db`.

**`NexusBookmark`** ‚Äî pure helpers.
- `create(for url: URL) throws -> Data` ‚Äî `url.bookmarkData(options: .withSecurityScope, ...)`.
- `resolve(_ data: Data) throws -> (url: URL, isStale: Bool)` ‚Äî `URL(resolvingBookmarkData: ..., options: .withSecurityScope, ...)`.
- `startAccessing(_ url: URL) -> Bool` / `stopAccessing(_ url: URL)`.

**`AppState`** ‚Äî `Codable` struct mapped to `~/Library/Application Support/com.nathantaichman.Pommora/state.json`. Holds **machine-specific** state that doesn't belong inside any nexus.
```swift
struct AppState: Codable {
    var schemaVersion: Int          // = 1
    var lastNexusBookmark: Data?    // security-scoped bookmark; nil = no nexus yet picked
    // v0.2+ adds: recentNexuses, lastWindowFrame, etc.
}
```

**`NexusIdentity`** ‚Äî `Codable` struct mapped to `.pommora/nexus.json` inside the nexus. Holds **nexus-portable identity** (no machine-specific data ‚Äî never holds bookmarks).
```swift
struct NexusIdentity: Codable {
    var schemaVersion: Int    // = 1
    var id: String            // ULID; stable across nexus rename/move
    var createdAt: Date
    // v0.2+ may add: per-nexus settings that should travel with the nexus
}
```

Nexus-specific *user* state (open tabs, sidebar collapsed state, active tab) lands in `.pommora/state.json` inside the nexus during v0.2+ ‚Äî separate from `nexus.json` (identity) and from app-level `state.json` (machine-specific). Three files, three concerns: identity (portable), app (machine), nexus state (portable, future).

**`FolderTree`** ‚Äî pure function: `func buildTree(at root: URL) throws -> SidebarNode` using `FileManager.default.enumerator(at:includingPropertiesForKeys:options:errorHandler:)`. Filters per the rules below. Sorted folders-first, alphabetical within each group.

**`SidebarNode`** ‚Äî `enum NodeKind { case folder, page, item }`; `name: String`; `url: URL`; `children: [SidebarNode]?` (nil for files, possibly empty for empty folders).

**`SidebarView`** ‚Äî replaces the v0.0 placeholder sections. Recursive `OutlineGroup` driven by `nexusManager.rootNode`. For v0.1: flat tree mirroring folders. The three-heading logical model (Spaces / Saved / Collections) lands in v0.8 ‚Äî handoff already flags this transition as a known gap.

**`SidebarRow`** ‚Äî `Label(displayName, systemImage: icon)` where `displayName = url.deletingPathExtension().lastPathComponent` for files, `url.lastPathComponent` for folders.

#### Data Flow

##### First launch (no `state.json` anywhere on disk)

1. App launches ‚Üí `ContentView.task { await nexusManager.loadOnLaunch() }`.
2. `NexusManager` checks for any prior bookmark ‚Äî finds none.
3. Calls `pickNexus()` immediately, presenting `NSOpenPanel`. Default location `~/PommoraNexus/` if it exists, else `~/`.
4. User picks `~/Desktop/MyTestNexus/` (or wherever).
5. Init flow:
   - Folder has `.pommora/` already? ‚Üí Load existing `nexus.json`, skip init.
   - Folder is empty? ‚Üí Silent init: create `.pommora/`, generate ULID, write `.pommora/nexus.json` with `{ schemaVersion: 1, id, createdAt }`.
   - Folder is non-empty without `.pommora/`? ‚Üí Confirm alert: "Initialize this folder as a Pommora nexus? Pommora will create a hidden `.pommora` folder for its index and configuration." Cancel returns to picker.
6. On confirmed init: create security-scoped bookmark from picked URL, write `~/Library/Application Support/com.nathantaichman.Pommora/state.json` with `{ schemaVersion: 1, lastNexusBookmark: <Data> }`.
7. Create `~/Library/Application Support/com.nathantaichman.Pommora/nexuses/<id>/`, mark excluded from backup via `URLResourceKey.isExcludedFromBackupKey`. Database file path is reserved at `nexuses/<id>/pommora.db` but NOT created in v0.1 ‚Äî that's v0.2's GRDB layer.
8. `NexusManager.currentNexus = Nexus(id: ..., rootURL: ...)`; `FolderTree.buildTree` runs; sidebar renders.

##### Subsequent launches

1. `loadOnLaunch()` reads `~/Library/Application Support/com.nathantaichman.Pommora/state.json` (`AppState`). If the file doesn't exist or `lastNexusBookmark == nil`, behave as first launch.
2. Resolve `lastNexusBookmark` via `URL(resolvingBookmarkData: ..., options: .withSecurityScope, bookmarkDataIsStale: &isStale)`.
3. Bookmark valid ‚Üí call `startAccessingSecurityScopedResource()`, walk into nexus, read `.pommora/nexus.json` (`NexusIdentity`) to get the nexus ID, resolve `nexuses/<id>/` in App Support, set `currentNexus`, render sidebar.
4. Bookmark stale (nexus moved on disk) ‚Üí re-create the bookmark from the resolved URL, save updated `AppState`, continue normally.
5. Bookmark resolution fails entirely (nexus deleted) ‚Üí alert user, present picker, treat as first launch.

##### Switch nexus (File ‚Üí Open Nexus‚Ä¶)

Trigger `pickNexus()` regardless of current state. On success, replace `currentNexus`, write new bookmark to `AppState`, save, rebuild sidebar.

##### Reset (Debug menu, DEBUG-only)

Delete `~/Library/Application Support/com.nathantaichman.Pommora/state.json`. Don't touch the nexus on disk or the per-nexus `App Support/nexuses/<id>/` directory. Next launch behaves like first launch.

#### Error Handling

| Failure | Behavior |
|---|---|
| User cancels first-launch picker | Show empty welcome state in main pane with "Choose Nexus‚Ä¶" button. Sidebar empty. App is usable for nothing else; no crash. |
| Bookmark resolution fails (nexus deleted) | Alert: "The nexus folder is no longer available. Pick a nexus to continue." ‚Üí present picker. |
| Bookmark stale (nexus moved) | Auto-refresh via `bookmarkDataIsStale: &stale`; if `stale == true`, re-create bookmark + save new pointer silently. |
| `.pommora/` exists but `nexus.json` is corrupt | Alert with the option to "Re-initialize" (writes a fresh `nexus.json` with a new ULID) or "Cancel" (returns to picker). |
| `Application Support` directory creation fails | Alert with the underlying error; app cannot function without this. |
| FileManager directory enumeration fails partway | Use `errorHandler:` closure to log + skip the failing entry, continue tree build. Don't abort the entire sidebar. |

#### Dual-Workflow Patterns: Dev Testing Nexus + Real Personal Nexus

The design supports running Pommora against two distinct nexus setups in parallel ‚Äî a dev testing nexus using existing on-disk content, and a separate real personal nexus for actual use.

**Dev testing nexus ‚Äî point Pommora at the Studio (or a subfolder).** Pick `/Users/nathantaichman/The Studio/Projects/Project Pommora/` to exercise the sidebar against the Pommora project's own document tree, or pick `/Users/nathantaichman/The Studio/` to see the full Studio structure. Pommora creates `.pommora/` at the chosen root.

Two practical caveats when using a git-tracked folder as a dev nexus:
- **Add `.pommora/` to `.gitignore`.** Pommora's per-nexus config + `nexus.json` shouldn't be committed to project repos. v0.1 implementation adds `.pommora/` to the Pommora project's own `.gitignore` automatically as part of the work; for any other dev-test nexus that's a git repo, Nathan adds it manually one time.
- **Hidden config folders are skipped from the sidebar** per the scope decision ‚Äî `.claude/`, `.git/`, `.obsidian/`, `.pommora/`, `.DS_Store` all stay invisible. So pointing at the Studio surfaces only the visible content (project subfolders, top-level `.md` / `.json`), keeping the test view focused on actual Pommora-relevant files.

**Real personal nexus ‚Äî pick a separate folder when ready.** Once Pommora is stable enough for real use, File ‚Üí "Open Nexus‚Ä¶" ‚Üí pick (e.g.) `~/Pommora/` or wherever personal notes live. The bookmark switches to the new nexus, the App Support per-nexus subdirectory keys off a fresh ULID, and the dev nexus stays intact in App Support (keyed by its own ULID) ‚Äî re-pickable later for continued testing.

**Running both in parallel during dev.** File ‚Üí "Open Nexus‚Ä¶" any time. Each nexus keeps its own `pommora.db` under `App Support/nexuses/<id>/`, so dev testing data and personal data never cross-contaminate. The active nexus is whichever one was last picked.
