#### Decision summary (locked end-of-session)

Pommora's Page editor is swapping from `Pallepadehat/MarkdownEditor` (CodeMirror 6 + WKWebView) to **Milkdown + Crepe** (ProseMirror + WKWebView). Driven by Nathan's smoke-test of Phase G â€” the Pallepadehat baseline + extensive CSS polish still didn't produce the Notion-like UI Pommora wants. Crepe ships that baseline out of the box.

**Locked choices for the swap:**

1. **Vendored, not SPM-dep'd.** The new editor wrapper lives as **source files inside Pommora's own tree**, not as a separate SPM package. Nathan needs to see every line of the JS + Swift in Pommora's own repo so he can inspect + iterate without round-tripping through a fork repo. Probable location: `Pommora/Pommora/PageEditor/` (Swift) + `Pommora/Pommora/PageEditor/web/` (JS bundle source).
2. **Crepe with default macOS theming first.** Use Crepe's `frame` theme (the most macOS-native of the three) as the baseline. **Custom styling packages** (Pommora brand accent, font tweaks, etc.) come AFTER the baseline lands and Nathan signs off on the default look.
3. **Stay with Live Preview / WYSIWYG editing model.** Crepe defaults work; no need to override.
4. **Defer Pommora-specific markdown extensions to later patches:**
   - `:::callout` + `@Columns` directives â†’ v0.2.9 (via `remark-directive`)
   - `[[wikilink]]` inline node â†’ v0.2.10 (5-component plugin pattern)
5. **Round-trip trade-off accepted.** ProseMirror's serializer normalizes stylistic choices (list marker `-` vs `*`, fence ``` vs ~~~, etc). Pommora's body changes consistently on save; external editors get stylistically-normalized files. Documented in `Features/PageEditor.md` post-swap.
