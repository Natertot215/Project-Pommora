#### Lessons Learned

- **Commit the WIRING with the feature, not just the new files.** The integration point (`CustomTableView(` → `ViewOutlineTable(` in `PageTypeDetailView`) was a modification to an already-tracked file; `e7719c0` staged only the untracked new files, so the table was never instantiated and we chased ghosts for hours. When adding a component, stage the call site that plugs it in — `git status` "clean" doesn't mean "wired." **→ candidate CLAUDE.md quirk**
- **Build-green ≠ wired ≠ verified.** The dup/config/Hide-Property fixes compiled and committed, but were never running because the renderer wasn't mounted. Verify on the actual running surface, not just the build.
- **Native-first, again.** Wrap the AppKit control (`NSOutlineView`) — disclosure, column resize/reorder/persist, alternating rows, keyboard nav come free; a SwiftUI hand-roll can't match the look/feel and re-implements all of it badly. Reinforces [[project-views-custom-table-failed-use-appkit]].
- **Don't merge in-progress work to `main`.** The accidental merge reintroduced the whole abandoned-custom-table lineage. `main` stays clean until the work is verified-clean; then one deliberate merge.
- **BUILD-DISPARITY is a known, recurring confusion flag — establish the SAME reference before claiming anything is broken OR fixed.** Over and over this session, Nathan reported a bug against a binary that predated the fix (a stale Xcode build still running while the fix sat unbuilt/unlaunched), or work was called "done" before it was on his screen — and hours went to arguing about a thing while we were literally looking at *different builds*. RULE: before either side asserts "broken" or "fixed," confirm the exact running binary maps to the source in question — `pgrep -x Pommora` → its build mtime vs the change's timestamp — and relaunch + confirm pid+mtime for Nathan every time. Controller-launched `open` builds and Nathan's Xcode `Cmd-R` builds are *different references*; name which one is running. **The 2026-06-13 reorder-persist "bug" was a textbook instance** — it was fixed in `1bf99e3`, but every failure report was against an older binary, and ~a full session was lost arguing about different builds. **RESOLUTION TECHNIQUE: a process-restart test settles a broken-vs-fixed dispute** — kill the app, set a *distinctive non-default* on-disk state, relaunch, and check whether the fresh process reflects it. A brand-new process reading the sidecar is the strongest re-entry check and is immune to stale in-memory / in-flight state. **→ candidate CLAUDE.md quirk** (twin of "build-green ≠ wired ≠ verified").
