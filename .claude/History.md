### Pommora ‚Äî History

Changelog + the home for locked decisions ‚Äî what shipped, oldest first. Brief by design; current state lives in the feature docs + `PommoraPRD.md`, roadmap in `Framework.md`, editor internals in `Features/MarkdownPM.md`. When an entry would enumerate file-level detail, it points to the canonical feature doc instead. The build splits in two eras: the **Swift origins** (a version ladder, condensed ‚Äî that build is archived under `Swift/`) and the **React continuation** (the live build at `React/`, where the detail lives).

#### Swift Origins ‚Äî the paradigm's first build

The whole domain paradigm was designed and versioned in a native SwiftUI build over roughly six weeks (2026-05-16 ‚Üí late June). It's paused, not deleted ‚Äî its docs are archived under `Swift/`, and its on-disk decisions carry into React unchanged. Mined to milestones:

- **v0.0.0 ‚Üí v0.1.0 ‚Äî foundation.** The shell (two-column `NavigationSplitView` + inspector, resizable panes), then the Nexus foundation: sandboxed folder picker, security-scoped bookmark persistence, `.nexus/` init, per-nexus App-Support subdir keyed by ULID. `.md` locked as the portability firewall after the editor-library exploration (Tiptap/WKWebView ‚Üí Milkdown ‚Üí native).
- **v0.2.x ‚Äî paradigm scaffolding + editor.** Every entity Codable + validator + `@Observable` manager, CRUD end-to-end under Swift-6 strict concurrency; `.trash/` foundation + CI. The native **TextKit-2** Page editor shipped (after WKWebView and Milkdown failed the visual bar), then the dynamic-syntax construct passes (markers reveal under the caret, standard CommonMark on disk) and Obsidian-parity Nexus folder adoption. Navigation (Pinned + Recents) shipped.
- **ParadigmV2 + Flat-Layout.** Vault became Pages-only; AgendaItem split into **AgendaTask + AgendaEvent**; per-kind sidecars moved to the nexus root (classified by sidecar filename, not folder wrappers). Locked: "Pommora" prohibited in on-disk schemas.
- **v0.3.0 ‚Üí v0.3.5 ‚Äî Properties + Connections.** The property system feature-complete: 11 `PropertyType`s, tagged `PropertyValue`, `PropertyDefinition` (stable ULID), `SchemaTransaction` (atomic multi-file commit), a GRDB SQLite index (rebuildable cache, files canonical). Then View-Settings chrome, a native IconPicker, **Contextv2** (user relations dropped ‚Äî the three context tiers are the only relation), and **Connections** ‚Äî `[[Page Title]]` end-to-end with a rename cascade and nexus-wide title uniqueness.
- **v0.4.0 ‚Üí v0.4.2 ‚Äî Contexts, Sets, Views.** PagePreview rebuilt as a real `NSPanel`; the three context tiers **decoupled into free-standing Areas / Topics / Projects** (no containment); the **Set** tier landed (schema-less folders inside a Collection); and **Views** shipped ‚Äî SavedView v2 feeding a pure filter‚Üígroup‚Üísort pipeline into Table + Gallery renderers, plus the grouping redesign (order modes, date buckets, empty-group handling).
- **Late-Swift hardening (06-18 ‚Üí 27).** A live **file watcher** (FSEvents, stamp-on-first-sight ULIDs for external `.md`s); the Nexus header + Homepage banner; a behavior-neutral codebase-health refactor (~1,300 tests green); **Collections / Sets / Sub-Sets** ‚Äî the Pages model collapsed to a schema-bearing Collection + a recursive Set nesting to any depth, sidecars unified to `_pagecollection.json` / `_pageset.json`; `modified_at` made frontmatter-canonical (mtime fallback) and optional-on-decode; and settings writes made symmetric read-modify-writes so Swift and the nascent React build stop clobbering each other's fields.

**Why the line moved to React:** SwiftUI friction plus Claude's large capability gap between Swift and TypeScript. React was scoped as a contingency, built to parity alongside late Swift, then became the primary build. See `CLAUDE.md` ¬ß Swift Origins.

#### React ‚Äî the continuation

The live build: an Electron desktop app, React + TypeScript renderer over a Node main process that owns the filesystem (architecture ‚Üí `Architecture.md`). It caught the Swift paradigm up on a new stack ‚Äî the on-disk format, domain model, and locked decisions all carry over ‚Äî and is where all current work happens. The entries below run oldest to newest.

#### React port ‚Äî parity on a new stack (2026-06 ‚Üí 07-01, branch `main`)

The Swift paradigm was rebuilt in Electron/React/TS behind thin swappable seams (SQLite behind `db.ts`, YAML behind `pageFile.ts`, IDs behind `ids.ts`, glass behind `Surface`), main-owned filesystem exposed through a narrow typed IPC bridge. The on-disk model is byte-compatible with Swift's (same sidecar keys, same `PropertyValue` tagging), so a nexus round-trips across both builds. **MarkdownPM** ‚Äî a CodeMirror-6 editor behind a swappable seam ‚Äî replaced the TextKit editor. The read path is read-only by construction; a single fs walk is the source of truth, SQLite returns later as a regeneratable accelerator. Detail ‚Üí `Architecture.md`, `MarkdownPM.md`.

#### PropertiesV2 ‚Äî definitions go nexus-wide (2026-07-01, branch `main`)

Property **definitions** moved from per-Collection sidecars to one nexus-wide registry at `.nexus/properties.json` (zod-validated, lenient read); a Collection's sidecar `properties` is a flat **assignment array** of prop-ids, joined ids‚Üídefs in `readNexus` so the renderer and view pipeline never changed. Values stay `prop_<ulid>`-keyed in frontmatter ‚Äî zero page migration. **Ratified decisions:** one fully-shared definition + options per property (divergent needs = a separate property, never forked options); Create validates names registry-wide while **Assign is an unvalidated, idempotent reference**; global **Delete** is the one fan-out op ‚Äî a timestamped recovery snapshot to `.trash`, then one atomic transaction strips every assigner; registry mutations serialize through a single chain (a review-caught lost-update race). **Agenda keeps its own `property_definitions`** (deliberate). Cross-Collection moves adopt-don't-strip by construction; cross-Collection queries are unlocked. Spec ‚Üí `Features/Properties.md`.

#### Tables ‚Äî the view renderer (Phases 1‚Äì2, 2026-07-01 ‚Üí 07-02, branch `main`)

The Table view became fully interactive across two phases. **Phase 1 (cells ¬∑ menus ¬∑ styles):** the gesture matrix ‚Äî title navigates, status/select/multi/context cells open the `PropertyPicker`, checkbox-look status cycles its group, numbers inline-edit, links/files open through sanctioned IPC, right-click always menus (per-type Style radios + Clear); per-view `column_styles` (per-key fold so a spread can't wipe siblings); the editing surfaces live table-agnostic in `PropertyEditing/` for Gallery/List reuse. The **Apple overflow model** (Nathan's live ruling): every column holds its resolved width, maxes uncapped, overflow h-scrolls the whole view past the pane. **Phase 2 (band drag):** group bands drag by their glyph on the sidebar's insertion-line gesture ‚Äî vertical reorder persists per-view, a Set band's reparent commits a real `moveSet`. The chip **hover-√ó** melt shipped here too: a Chromium dropped-repaint family forced an opacity-only reveal mechanism (three text copies, crisp/melt/blur twins) ‚Äî laws + re-verify matrix in `Guidelines/Build-Gotchas.md`. Spec ‚Üí `Features/TableView.md`.

#### Table band order ‚Äî manual-only, view-owned (2026-07-02, branch `main` ‚Äî LOCKED)

Group-band ordering is **manual-only** ‚Äî no user-facing order mode. Bands *default* to the defined order (property options in sequence; status upcoming‚Üíin-progress‚Üídone; structural bands seed from fs `set_order`), and the **first drag snapshots that default into an owned sequence that never re-derives**. Homes: structural ‚Üí the view-level `group_order: string[]` (one flat set-id array across every nesting level); property ‚Üí `group.order` + `order_mode: 'manual'`. **Structural bands do not mirror the sidebar** ‚Äî vertical reorder is per-view and never touches the filesystem; only a cross-tree Set reparent commits `moveSet` (carrying the destination's current fs child order + the moved id appended ‚Äî the order-leak guard). **No "None" band** ‚Äî value-less pages render as the loose-row tail. Reversal seams kept both ways; no migration. Mechanics ‚Üí `Features/TableView.md` ¬ß Groups.

#### PommoraIcons + glass chrome (2026-07-02, branch `main`)

The icon set became **PommoraIcons** (`design-system/symbols`): **Lucide** primary, **Tabler** the opt-in second source at 1.75-stroke to fill gaps; shipped pre-IconPicker so the picker enumerates it from day one. Bannered container/context views gained the entity's glyph beside the title (banner ‚áí icon; pages and homepage stay title-only ‚Äî Nathan's call), hosting the shared `DetailTitleHeader` (native Rename / Change Icon menu). The nav + settings dropdowns adopted a **`NotchedPane`** beak reshaped to the Apple-popover silhouette (one path serves as frost clip AND SVG-stroked outline). Gotcha reconfirmed: the Bloom class rides the pane + frame individually ‚Äî an opacity-animated wrapper backdrop-roots the frost to nothing (`Build-Gotchas` ¬ß Glass). Registry ‚Üí `symbols/Symbols.md`; assignments ‚Üí `Features/Icons.md`.

#### ViewPane Properties Flow ‚Äî the full assign surface (2026-07-02, branch `main`)

The Properties pane became the whole assign surface (the 7-2 ratified log, built T1‚ÜíT7 attended): assigned rows over a bottom-pinned **All Properties** disclosure that **rises** open (an elastic spacer collapsing on the pane's beat) to list every unassigned registry def with a `+` promote; a **nexus-wide cosmetic property order** persisted in `.nexus/properties.json` (grown to `{order, defs}`, lenient legacy read); the two-region drag (region-owned classification ‚Äî reorder-assigned / reorder-nexus / assign-at-slot / unassign); row right-click **Rename** (a net-new `renamingProperty` store channel) **¬∑ Remove**; the editor header's pane-gated **Delete** behind main's native confirm. Every pane push at every depth slides (nested PaneSlider). **Remove now strips every member page's value and caches it restorably on the Collection sidecar** (one atomic transaction); re-assigning restores per-value against the def's CURRENT type + options; Delete purges caches everywhere ‚Äî no dormant foreign values (a reversal of the 07-01 leave-in-frontmatter ruling). **Duplicate names allowed on create AND rename** (flat policy ‚Äî ids keep twins safe); **Max Properties killed**. Spec ‚Üí `Features/Properties.md`.

#### Visibility pane ‚Äî the first ViewPane sibling (2026-07-03, branch `main`)

The ViewPane's Visibility entry went live (`HiddenPane`), spec'd live with Nathan across three in-session respecs and verified via CDP against throwaway collections seeded with a rich temp-property set ‚Äî rest, eye hover-swap, hide/unhide, reorder-among-many, multi-hide, drag-down-to-hide, drag-up-to-unhide into a mid-list slot, contexts-inert, the fresh-container single-view path ‚Äî all screenshot-evidenced, every artifact deleted and the Nexus git state diffed byte-identical after. **Locked shape:** a static **Contexts** block on top (fixed Areas ¬∑ Topics ¬∑ Projects order; hiding a context **ghosts it in place**, never relocating it), then the properties as **one heading-less list** ‚Äî shown rows in view order, hidden rows ghosted after them in collection order; the ghost is the boundary. **Title appears nowhere** (can't hide; reorder belongs to column drag). **Drag language:** into the shown zone is positional (drop line ‚Äî reorder, or unhide at the slot); a shown row into the hidden zone is a **membership drop that hides** (area highlight, no line ‚Äî the hidden order is derived, never authored). **Ordering law:** hide = flag-only (`hidden_properties`) so `property_order` remembers the slot and an eye-unhide restores it; only a drag-in chooses a new slot; the shown zone is a **window into the full column order**, so drops translate through the successor anchor and Title/tiers hold their absolute slots. Eyes read secondary + ghost, un-ghosting on hover (glyph swaps open/off), tertiary on hidden rows. A post-green breaker round folded three findings (the `view_default` sentinel adopted into the per-machine active-view pointer so a second surface can't mint a rival; a dragged hidden row's double-dim; the all-hidden empty shown-zone target, accepted). `PaneDnd` gained an injectable `slot` rule and now owns the shared `RowShell`; `TableView` gained a **confirm-clear** so an external writer renders live instead of being masked. Spec ‚Üí `Features/Views.md` ¬ß Visibility Pane.
