#### App-Wide Auto-Scroll (2026-07-14)

Every drag's edge-scroll collapsed onto **one shared primitive** — `interactions/autoscroll.ts`, a singleton rAF loop each drag feeds — replacing two duplicated per-surface copies (the engine's and the editor block-drag's) and retrofitting the three surfaces that never had it (sidebar, table rows, table bands). **Locked — one fixed scroller resolved once at drag start, not per-frame:** the ratified design rejected the general-library "scroller under the pointer each frame" (`elementsFromPoint`) because no core Pommora drag crosses scroll containers — each lives in exactly one — so a scroller resolved once (explicit for the drags that self-compensate the scroll delta or scroll the CM `scrollDOM`, else the **axis-aware `findScroller`**) is simpler and dodges a bug class (viewport-exit stall, token-cache incoherence); the lone crosser, the cross-list board, stays a prospect precisely because it'd need the rejected per-frame path back. **Locked — the loop owns the scroll, the pointer only feeds a point:** it scrolls every frame off the last point (holding still at an edge keeps scrolling), in px/sec × frame-delta with sub-pixel accumulation (frame-rate-independent, no ProMotion double-speed) and a dt clamp so an rAF stall can't teleport the scroll, plus two feel wins — **distance-based acceleration** (a scroll run eases in from a non-zero floor and climbs to a ceiling with the distance it covers, so a longer drag-scroll goes slightly faster; resets on leaving the band) and **direction-intent** (a direction won't scroll until the pointer leaves that band once, killing the grab-at-edge rocket). **Locked — token-driven off the drag element:** the four tunables live in `interactions/autoscroll.css`, read off the drag element once at start and cached, so a surface overrides them on itself or an ancestor. **Locked — the module owns a termination backstop** (blur / visibilitychange / pointercancel) that stops the *loop only* (each surface keeps its own gesture abort), and `startAutoScroll` returns an **instance-scoped stopper** so a bystander surface's unmount can't halt a live drag. The **axis-aware `findScroller`** was the enabling fix: a vertical table drag must skip the x-only `.table-view` to reach `.detail-scroll`. Consumers: engine · SurfacePM · settings pane · block-drag · sidebar · table rows · bands — no second copy remains. Ten-task inline build, each task adversarially attacked before the next (the loop's dt-teleport, a cross-Zone stop, a table unmount-leak + ungated per-frame measure all caught and folded); gates green throughout. **Prospects** (deferred, each named with its real cost): MarkdownPM list drag, table column horizontal reorder, GFM-table drag, grouping pane, the cross-list board. Decision log + plan in `Planning/`; the primitive → [[PommoraDND]] §II. Autoscroll.

#### The Block Surface — SurfacePM (2026-07-10 → 07-13)

Pommora's composable dashboard layer landed: a **BlockHost** renders a mosaic of draggable, resizable tiles holding real content over the in-house **SurfacePM** tessellation engine ([[PommoraDND]]'s sibling), the Homepage serving as the removable dev host. **Locked — host-agnostic (D-2):** the surface works identically under every resolution of the contexts question, so it ships *before* any real host does; the block document (SurfacePM's `layout` split-tree + a tagged-union `blocks` array + a host lock) loads per-host on open, never in the tree walk, and the watcher ignores host content folders so a block edit never costs a re-walk. **Locked — repair-not-reject at every level (E-1):** unknown/foreign tile entries are preserved and rendered inert (never stripped, never crashing the host), a broken hand-edit repairs without wiping the document's survivors, dead references render inert — and every write is a locked read-merge-write touching only its own keys, so foreign keys survive by construction.

Three **tile types** — a **markdown block** (file-backed prose, a ULID-named `.md` in the host folder, a connection *source* only), a **page embed** (a reference to a real Page), and a **view embed** (a container's saved view or a block-owned config, copied at pick time, never synced back). **Locked — the embed IS the CM6 view (G-11):** one seam renders a Page inside any foreign surface as a read-only portal carrying every MarkdownPM affordance, editability flipped in place through a live-reconfigured compartment (a facet change, never a remount) — an embed edit is a page edit through the page's own debounced save. Two framework laws with reach beyond blocks: **popups escape the tile** (a `transform`ed ancestor re-anchors `position:fixed`, so any popup born in an embed portals to body) and **scroll is caret-priority** (blocks are wheel-transparent at rest; only the caret-holding block scrolls internally).

**Surface interaction:** right-click creates (wedge-fit under the click, or a full-width band on open background); the notched **drag handle** opens the block menu (Type ▸ · Style ▸ · Scale ▸ · Remove); window-style edge/corner **resize** magnetizes to neighbors, with interior holes impossible by construction; a geometry-only **host lock** freezes position/size while content stays live; and an embedded page signals itself with an accent border. **Locked — per-block Scale is view-agnostic freeze-inset (G-10):** five discrete steps ride one inherited `--block-zoom` var (a registered animatable `@property`, transitioning on the standard beat) set from an accent-outlined dropdown off the handle menu; the tile-edge inset + drag handle + resize edges always stay fixed (manipulation chrome, not content), while a markdown/page tile scales only its text + editor glyphs (structure frozen) and a view tile scales its table as a unit (the grid's own CSS zoom compounds the Scale over the base density) — an off-grid stored factor snaps to the nearest configured step (the view tile's Scale animates on the same beat, its grid relayout bounded to the one-shot transition). **The accent (accent @ tint-secondary) marks an embedded page's identity** — its tile border on the surface and the openable page+location field in its handle menu — while the surface's own pickers stay plain glass. Gates green through the arc (typecheck · 1492 tests · build); the Scale slice was adversarially reviewed (0 High/Med) and simplifier-cleaned. Spec → `SurfacePM.md` (+ [[PommoraDND]]); decision provenance in `Planning/`.

#### View Filtering — the Filter Engine (2026-07-10)

The view **filter engine** shipped to main; the authoring **FilterPane** was built on the Grouping/Sorting chassis, driven through a long live-UIX loop, then **pulled before merge — the functionality is right, the pane's design wasn't** (blank leaf on both doors, a redesign parked in Pending). What landed is the engine — the pipeline applies any stored `filter` with the full type-aware matrix. **Locked — every new operator is three coordinated changes:** the `FILTER_OPS` registry entry (the `FILTER_OP_SET` gate no-op-passes an unregistered op — a filter that looks applied but does nothing), the per-type evaluator branch, and the per-op operand semantics — `contains_any` guards its empty operand set explicitly (`[].some()` would blank the table), date `is` truncates both sides to the calendar day (never exact ms), `starts_with` is case-insensitive. New ops: `starts_with · contains_all · contains_any · is_before · is_after · greater_or_equal · less_or_equal · is_inside · is_not_inside`. **Locked — Title, Location, and Context became filterable:** Title routes to the text matrix (inverting its pinned no-op test), Context splits off the file presence arm into `evaluateList` membership, and **Location filters any-depth** via a descendant-id Set precomputed **once per operand** (the exported `subtreeIds` walk, `setTree` threaded into `applyFilter`) and membership-tested per row — never a per-row ancestor walk; a dead set id is a no-op pass. **Locked — the operand model:** `FilterRule` gained `values?: string[]` for multi-operand ops (any-of / none-of on single-valued options, Is Any / Is All on arrays), and `MATCH_MODES` widened to `all | any | none` — a root `none` skips filtering (the pipeline early-returns) while the rules persist, so "disabled" is a real on-disk state. The no-op-pass rule is pervasive (unknown op, dead property, dead set id, missing operand all pass, never exclude), with one deliberate Swift-parity exception: an id-list membership test with a missing *single* operand is false. Refactor kept: `contextOptionsFor` extracted to a shared `pipeline/contextOptions.ts` (TableView repointed). **The pane's spec is preserved for the redesign** — its ratified decision log (flat rule rows serializing onto the recursive `FilterGroup`, the Matches header + per-row And/Or connectors, the shape-based lock predicate, the stay-open chip picker) survives in `Planning/`, certified across 3 adversarial rounds and hardened by a shipped-code review (which caught a locked-state data-loss hole + a null-byte accumulator delimiter). Spec → `Views.md`; decision log + plan in `Planning/`.

#### The Watcher Walk Goes mtime-Gated (2026-07-06)

#### Page-Write Lock Unification (2026-07-04)

Every write to a page `.md` now serializes on ONE per-file lock. The schema-op page cascades (Select/Status option rename·remove·clear, the `[[link]]`-rename + tier-unlink cascades, property Delete and Remove/restore) had ridden `SchemaTransaction` on a *separate* lock from the cell-write path's `serializeOnFile`, so a cascade racing a table-cell edit on one page could silently clobber a value (the F1 race). **Locked — one lock for all page writes:** `serializeOnFile` (hoisted to `io/fileLock`, reading fresh INSIDE the lock so no stale snapshot lands) now guards the cascades, `setProperty`/`setTier`, the editor-body autosave, and the banner write alike; `SchemaTransaction` stays only for Agenda (pure JSON, never on the cell path). Cross-page atomicity traded for per-file recoverability — Delete keeps its `.trash` snapshot, Remove keeps cache-before-strip. **Locked — the lock key is the canonical path:** `openSession` realpaths the root so the cascade and cell-write keys match on symlinked-root ancestries (an adversarial round caught them silently diverging on `/var`→`/private/var`); canonical stays on the in-process locks, while persistence + display keep the RAW user path (iCloud-`~/Documents` robustness).
