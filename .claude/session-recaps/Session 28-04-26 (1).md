## Architectural constraint surfaced

**Drag-reorder is deferred.** Finder-style displacement drag (source dims in place, gap at drop point, no blue bar) is architecturally blocked in `.listStyle(.sidebar)` — `NSOutlineView` takes over and renders the legacy blue insertion bar. The macOS 26 container-drag API only works correctly under `.listStyle(.inset)`.

This is a v1.1 problem. Current app uses the original string-prefix drag (`"folder:UUID"`, `"file:UUID"`) with `withAnimation(.snappy)` live reorder on hover — the walking skeleton behavior.
