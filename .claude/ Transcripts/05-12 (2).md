#### Session Insights (Nathan)

1. **Figma → React:** Figma produces really good UIX for React and the translation workflow is a real, supported path. It's also gimmicky in places, requires tweaking on things that look obvious to a designer, and Claude has trouble implementing some things directly. It's a real option that gives Nathan the design system he wants — not a free one.

2. **SwiftUI native editor:** A native markdown editor on `AttributedString` is a real possibility for Swift Pommora — build it ourselves or fork an open-source one. More work than simply integrating BlockNote / TipTap on React, but the result is fully native. Research done independently by Nathan alongside this session.

3. **Stack decision reframe:** Will likely come down to what can be achieved with the text editor, and whether easier React integration is worth the harder build process for the rest of the app (where UI responsibility falls on Nathan rather than adapting Swift's existing UIX). Live React demo is the gate — the Figma file reveals static intent; the working app reveals interaction behavior.
