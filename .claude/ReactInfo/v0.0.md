### v0.0 — App Opens to Styled Three-Pane Shell

The first build. Goal: prove the toolchain and architecture by getting Pommora to launch into a styled, token-driven three-pane layout. No interactivity, no data wiring.

> **⚠ Stack-locked spec.** This document assumes the React + Electron path. The stack is currently under re-evaluation (see `PommoraPRD.md` "Stack — Under Active Evaluation"). If the SwiftUI path is chosen, this entire spec is replaced by a SwiftUI scaffold spec (NavigationSplitView three-pane shell + token-derived Color/Font extensions). Do not begin implementation until the stack decision is confirmed.

#### Prerequisite — design system (pre-v0.0)

The Figma design-system build is complete before v0.0 begins. Build brief: `// Planning//Figma Prompt.md`. Tokens, primitives, composed components, and the three-pane shell mockup all exist in Figma. v0.0 builds the React shell that **consumes** the design system; it does not design the design system.

#### Build order (if React + Electron is chosen)

1. **Install toolchain** — Electron, Vite, React, TypeScript, Tailwind v4, `react-material-symbols`, `better-sqlite3`, `electron-rebuild`, `electron-builder`. Get the empty scaffold compiling and launching a blank Electron window.

2. **Export tokens from Figma** — Figma Variables export to `tokens.css` as CSS custom properties on `:root` (e.g., `--accent-primary-active: #A78BCC`). The exported file lives at `UI-UX// Design// tokens.css` and is checked in.

3. **Build primitives + composed components** — translate the Figma component library into React + Tailwind under `UI-UX// Components//`. Components consume tokens via CSS custom properties (no hardcoded values, ever). Material Symbols wired through the semantic role indirection layer (`.pommora// symbols.json`, seeded from the icon role table in the Figma Prompt; placeholder `crop_free` used until specific icons are finalized). Iterate on Pommora's own Vite + Electron dev server (root: `UI-UX// Components//`). The localhost preview IS the component gallery; **no Storybook intermediary**.

4. **Build the v0.0 shell** — wire the three panes plus the top-bar tab chrome from the composed-component library. Verify against the acceptance criteria below.

---

#### Goal

`npm run dev` launches the Electron app. Window opens to a three-pane shell — sidebar (left), main content (center), inspector (right) — styled with design tokens. Window is resizable. No console errors. Closing the window quits the app on macOS in standard fashion.

#### Toolchain

- **Electron** (latest stable; bundles Node 22+)
- **Vite** (renderer bundler with hot-reload)
- **React 18+** (UI)
- **TypeScript** (strict mode)
- **Tailwind CSS v4** (styling, with CSS custom properties from the design system)
- **`react-material-symbols`** (icon system; outlined style as default)
- **`better-sqlite3`** (installed but not used yet — verifies native module setup works)
- **`electron-rebuild`** or equivalent (for native modules in Electron)
- **`electron-builder`** (configured but not used to build a binary yet)

#### Project Structure

Pommora doesn't enforce the earlier Core / Adapter / UI layer separation. The v0.0 scaffold has the two processes Electron requires (`src//main//` for the Electron main process; the renderer entry in `UI-UX//Components//`) plus a shared types folder for IPC contracts. The renderer lives outside `src//` so the design system (`UI-UX//`) is a peer of the app code, not a subfolder — keeping the component library and the localhost dev server together.

```
Pommora//
  package.json
  tsconfig.json
  vite.config.ts                  ← root: './UI-UX/Components' (wired in step 3)
  tailwind.config.ts
  electron-builder.yml
  src//
    main//                        ← Electron main process (Node)
      index.ts
      window.ts
    shared//                      ← Shared types (IPC contracts, etc.)
      ipc-types.ts
  UI-UX//                         ← Design system home (peer of src//)
    UI-UX.md
    Design//
      Design Guidelines.md
      tokens.css                  ← CSS custom properties (populated in step 2 from Figma export)
    Components//                  ← React component library + Vite renderer + localhost
      Component Guidelines.md     ← only file that pre-exists; everything below lands during step 3
      (index.tsx, App.tsx, shell//, primitives//, styles// all populated by the
       Figma → code translation step — NOT pre-scaffolded)
  .claude//
    (project instructions, planning, history, etc.)
```

The `UI-UX//Components//` folder exists with only `Component Guidelines.md` pre-v0.0; component code (primitives, composed components, the renderer entry, shell, styles) all land during step 3's Figma → code translation, not before. Pre-populating the folder with skeleton files would defeat the cascade discipline (every component is born from Figma, never invented in code first).

#### Three-Pane Shell

| Pane | Default width | Resizable | Surface tier | Contents (placeholder) |
|---|---|---|---|---|
| Left — Sidebar | 240px | Yes (drag splitter) | `surface/secondary` | "Sidebar" label + 3 placeholder nav items with Material Symbol icons |
| Center — Main Content | flex-1 | (auto, fills remaining) | `surface/primary` | "Pommora" heading; "Welcome to v0.0" subtitle |
| Right — Inspector | 280px | Yes (drag splitter) | `surface/secondary` | "Inspector" label + placeholder text "No selection" |

Both side panes are resizable via drag splitters from v0.0 onward. Default widths (240 / 280) come from the proposed shell sizing in `PommoraPRD.md`. Persisting resized widths across launches is in scope for v0.0 (use `localStorage` or `window-state` equivalent).

**Top-bar tab row** sits at the top of the Center pane (above the placeholder content). In v0.0 it renders as visual chrome only — a single placeholder tab (active, labeled "Welcome") plus the `+` new-tab button at the trailing edge. The `×` close button and click-to-switch behavior are not wired (those land in v0.1 when files start opening). The tab row uses the same surface tokens as the rest of the shell; the active tab is visually continuous with the main pane it heads (`surface// primary// bg`).

#### Design Tokens (Source: Figma export)

Tokens come from the Figma Variables export (per `// Planning//Figma Prompt.md`), not from hand-written values. The exported file is stored at `UI-UX// Design// tokens.css` and is checked into the repo. Components in `UI-UX// Components//` import it through `styles//index.css`.

Sample shape (illustrative — canonical token names and baseline values live in `// Planning//Figma Prompt.md`):

```css
:root {
  /* Surfaces */
  --surface-primary-bg: #1C1C1D;
  --surface-secondary-bg: #191919;

  /* Text */
  --text-primary: #F1F1F1;
  --text-muted: #D5D5D5;

  /* Accent (single-hue purple, 2x2 matrix) */
  --accent-primary-active: #A78BCC;

  /* Typography */
  --font-family-sans: -apple-system, "SF Pro", system-ui, sans-serif;
  --font-family-mono: "SF Mono", ui-monospace, monospace;
  --font-size-body: 14px;
}
```

Variable names follow the dual-export naming convention (`surface// primary// bg` ↔ `--surface-primary-bg`). If the SwiftUI path is chosen, the same Figma Variables export to SwiftUI `Color` and `Font` extensions instead; variable names stay constant across stacks.

#### Acceptance Criteria

1. `npm install` completes without errors (including native module compilation for `better-sqlite3`)
2. `npm run dev` launches the Electron window
3. Window displays the three-pane shell using the token values above
4. Window is resizable; sidebar and inspector are drag-resizable via splitters; main pane flexes to fill the remainder. Resized widths persist across launches.
5. Material Symbol icons render in the sidebar (placeholder `crop_free` for any role not yet finalized)
6. Top-bar tab row renders above the main pane with one placeholder tab + the `+` button (visual only; no click handlers wired)
7. Zero console errors, zero React warnings
8. macOS-standard window controls (red/yellow/green) work correctly
9. Cmd+Q quits the app cleanly

#### Out of Scope for v0.0

- Any interactivity beyond window resize
- Tab interactivity (click handlers, open / close, reordering, persistence) — tab chrome renders but isn't wired; functionality lands in v0.1
- File watching, vault reading, SQLite usage
- Editor library integration (BlockNote or Tiptap)
- Real content, real navigation
- Light/dark toggle
- Settings or preferences UI
- Code Connect with Figma
- Production build / packaging

#### Verification Steps

After scaffolding:
1. Run `npm install` — confirm `better-sqlite3` native module compiles
2. Run `npm run dev` — confirm window opens
3. Inspect window — confirm three panes visible with token-driven styling
4. Resize window — confirm panes behave correctly
5. Open DevTools — confirm zero errors / warnings
6. Test Cmd+Q — confirm clean quit

If all pass, v0.0 is done. Move to v0.1 (vault reads) per `Framework.md`.
