### v0.0 — App Opens to Styled Three-Pane Shell

The first build. Goal: prove the toolchain and architecture by getting Pommora to launch into a styled, token-driven three-pane layout. No interactivity, no data wiring.

> **⚠ Stack-locked spec.** This document assumes the React + Electron path. The stack is currently under re-evaluation (see `PommoraPRD.md` "Stack — Under Active Evaluation"). If the SwiftUI path is chosen, this entire spec is replaced by a SwiftUI scaffold spec (NavigationSplitView three-pane shell + token-derived Color/Font extensions). Do not begin implementation until the stack decision is confirmed.

---

#### Goal

`npm run dev` launches the Electron app. Window opens to a three-pane shell — sidebar (left), main content (center), inspector (right) — styled with design tokens. Window is resizable. No console errors. Closing the window quits the app on macOS in standard fashion.

#### Toolchain

- **Electron** (latest stable; bundles Node 22+)
- **Vite** (renderer bundler with hot-reload)
- **React 18+** (UI)
- **TypeScript** (strict mode)
- **Tailwind CSS v4** (styling, with CSS custom properties from the design system)
- **`react-material-symbols`** (icon system; outlined style as default)
- **`better-sqlite3`** (installed but not used yet — verifies native module setup works)
- **`electron-rebuild`** or equivalent (for native modules in Electron)
- **`electron-builder`** (configured but not used to build a binary yet)

#### Design Tokens (Placeholder Values)

Tokens follow the dual-export naming convention so swapping in real Figma values later is a value change, not a rename. Stored in `src//ui//styles//tokens.css`:

```css
:root {
  /* Surface tiers */
  --surface-primary-bg: #1a1a1a;
  --surface-primary-fg: #e8e8e8;
  --surface-secondary-bg: #141414;
  --surface-secondary-fg: #c4c4c4;

  /* Borders */
  --border-subtle: #2a2a2a;
  --border-strong: #3a3a3a;

  /* Text */
  --text-primary: #e8e8e8;
  --text-muted: #888888;

  /* Type scale */
  --text-base: 14px;
  --text-lg: 16px;
  --text-xl: 20px;

  /* Spacing rhythm (4px base) */
  --space-1: 4px;
  --space-2: 8px;
  --space-3: 12px;
  --space-4: 16px;
}
```

These get replaced with real values from the Figma export when the design file matures. Variable names stay constant.

#### Acceptance Criteria

1. `npm install` completes without errors (including native module compilation for `better-sqlite3`)
2. `npm run dev` launches the Electron window
3. Window displays the three-pane shell using the token values above
4. Window is resizable; panes adjust correctly (sidebar/inspector fixed widths, main flexes)
5. Material Symbol icons render in the sidebar
6. Zero console errors, zero React warnings
7. macOS-standard window controls (red/yellow/green) work correctly
8. Cmd+Q quits the app cleanly

#### Out of Scope for v0.0

- Any interactivity beyond window resize
- File watching, vault reading, SQLite usage
- BlockNote integration
- Real content, real navigation
- Light/dark toggle
- Settings or preferences UI
- Code Connect with Figma
- Production build / packaging

#### Verification Steps

After scaffolding:
1. Run `npm install` — confirm `better-sqlite3` native module compiles
2. Run `npm run dev` — confirm window opens
3. Inspect window — confirm three panes visible with token-driven styling
4. Resize window — confirm panes behave correctly
5. Open DevTools — confirm zero errors / warnings
6. Test Cmd+Q — confirm clean quit

If all pass, v0.0 is done. Move to v0.x (Core layer scaffolding).
