### Session — 05-09 (Foundation Lock-In)

#### What we started with

Clean slate. The project's `.claude//` folder contained only template stubs (`CLAUDE.md`, `HandoffMD.md`, `HistoryMD.md`, `FrameworkMD.md`) populated with `[bracketed placeholders]`. No code, no Figma file, no architecture decisions. Nathan brought in a broad vision document framing Pommora as a hybrid Notion+Obsidian personal management platform — Obsidian's local-first openness plus Notion's database/view capabilities — with three open questions on the table: SwiftUI vs React, what pages actually are at the data layer, and whether to abandon local-first for cloud convenience. Session opened via `/superpowers:brainstorming` to drive an idea-to-spec arc.

#### What happened along the way

A long, oscillating brainstorm that cycled between two stacks before landing. Established early that Pommora is personal-first but always open-source — architecturally allow future multi-user/cross-device, but don't pay for it now.

Storage model emerged through several iterations. Nathan's first load-bearing constraint: *"having Claude Code understand the system is very important to me; less abstract than Notions MCP reliance; similar to obsidians local-first ease."* Recommended file-canonical Markdown + frontmatter + SQLite index. He pushed back on Markdown's expressiveness limits for callouts, columns, and inline views, then resolved it himself: *"the readability of that to Claude via markdown isnt a priority; claude could just see 'inline view to XYZ database' as the actual syntax it produces."* That unlocked the fenced-directive approach (`:::view source="Tasks" type="board" :::`) — file shows the spec, not the rendered data.

Stack decision oscillated. Initial recommendation: React + Electron + BlockNote, citing block-editor ecosystem maturity. Then Nathan reframed: *"Im not compitent in Coding, I dont know how to code, and this is my first actual project... If we go with Swift, then i would lay down the UI level customization emphasis such as colors and icons -- apples system looks great to me."* Reversed recommendation toward SwiftUI on cohesion grounds. Researched Swift ecosystem — discovered iOS/macOS 26's first-class `AttributedString` rich text in `TextEditor` (WWDC 2025) closes the worst of the editor gap. Then Nathan called out the underlying issue: *"these editor things and design system time estimates are ridiuclous cinsodering YOU Will be doing the work using modern capabilities."* Pulled current Claude Opus 4.7 benchmarks (87.6% SWE-bench Verified, 60% reported time savings on SwiftUI, full Xcode 26.3 Agent SDK integration) and recalibrated all editor estimates from "months" to "weeks of active sessions." Time-to-v1 gap between stacks effectively collapsed.

Figma access surfaced and further deflated the design-system concern. Final call: *"lets go forward with this as if were commiting to React + Electron using Tailwind. But also ensure that we dont go beyond the point where the core architecture and funcionality couldnt be mirrored into a swiftuI interface without much complexity."* Ports-and-adapters architecture established: Core (pure TS) / Adapter (thin TS bridge) / UI (React + Tailwind + BlockNote), with Core having zero React/BlockNote/Tailwind/Electron-renderer imports.

Plan mode entered for documentation infrastructure plan; approved. Mid-execution Nathan stopped: *"BlockNote is not locked in yet, neither is SQlite3; I still need to understand what that actually means."* Spent meaningful time explaining Electron's main/renderer split, SQLite as a single file (not bound to one machine), and why a database is necessary at scale. Cloud concern surfaced: *"Id just want a system where I have both markdown files, the necessary databse for databases, tagging and properites, and also the ability to have all of that be exported on the cloud If I ever wanted to do a sync feature."* Refined schemas to live in `_pommora//schemas.json` (file, not SQLite-only) so the entire nexus is cloud-export-ready by zip. Settled: better-sqlite3 + Tailwind + Material Symbols (modern variable-font version) all locked.

Final direction: *"dont repeat 'MD' in the Framworkd, Handoof,, ect... thats what I call it -- MD = .md; change the names"* and *"Keep what weve commited to breif and light for now, this is only just the beginnijng."* Renamed `HandoffMD.md` → `Handoff.md` etc., wrote all docs tight.
