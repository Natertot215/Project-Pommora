**Design-system build brief (the big one).** Nathan asked for a prompt template that captured all the architectural locks plus the placeholders he'd fill in. After a wave of clarifying questions, the visual direction locked: Notion-comfortable density; pastel-leaning muted color treatment; SF Pro + SF Mono (system-native); flat dark chrome (no shadows except on overlays); mixed-scale rounding (pill for tags, tight for buttons / toggles / labels, surface for cards / panels / modals — Notion / Claude reference).

Concrete baseline values came next: surfaces `#1C1C1D` (primary) / `#191919` (secondary); text primary `#F1F1F1`, muted `#D5D5D5`; em-relative heading scale (H1 2.0 → H5 1.0; no H6); body 14px baseline. Accent: a single-hue purple in a 2×2 matrix (primary / secondary × active / muted) — pastel-muted, descending in saturation + lightness. **End-of-day construct-level wave.** Nathan came back with three locks that needed propagation:

1. **Toggles aren't a separate directive.** Heading-fold is built-in UI behavior on every heading, not a `:::toggle` construct. Markdown-directive count dropped from 2 (`@Columns` + toggles) to 2 (`@Columns` + new `:::callout`). Removed `:::toggle` everywhere.
2. **Callouts ≠ blockquotes — they're distinct constructs.** Callout = outlined-box, minimally rounded, primary text color, independent border token. Blockquote = filled background with left-side emphasis bar; the bar defaults to the text color of the quote, NOT accent.
3. **Code tokens.** `code// fg` defaults to `#FF2525` (tied to color primitives); `code// bg` defaults to `#323233`; mono font (SF Mono) at 1.0 em.

Also locked: tab bar height 35px default. Sidebar section headings (Spaces / Saved / Collections) are text-only headings — no icons; entity-kind icons (page / item / collection / space) appear on navigation rows inside each section. And:

> "dont assign icons to the figma prompt; establish the placeholder untill Nathan decides"

Stripped the Material / SF columns from the icon role table entirely. All roles default to the `crop_free` placeholder until Nathan reviews per-role candidates. Embedded the actual `crop_free` SVG path in the placeholder convention block so a fresh session can render it exactly.

Did a final sweep across all docs (Pages, PRD, Framework, History, CLAUDE, Handoff, Architecture, Domain-Model, ReactInfo, Resources, SwiftInfo, Figma Prompt, UI-UX guidelines) — cleaned every stale `:::toggle` reference, "callouts are styled blockquotes" framing, and pre-assigned-icon mention. Only legitimate remaining references: the Toggle switch primitive (UI control), "Toggle sidebar visibility" (verb usage on chrome buttons), and PRD's explanatory note that there is no separate `:::toggle` construct.
