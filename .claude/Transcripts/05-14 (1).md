#### Arc

The session opened with a plan to ship Sidebar.md's selection-language spec (subtle gray fill + accent foreground, matching Mail's "Favorites" reference). The plan made three changes: delete `.listRowInsets` overrides on flat rows; add a `SidebarLabel` wrapper reading `@Environment(\.backgroundProminence)` to flip foreground color on selection; apply `.tint(Color(nsColor: .unemphasizedSelectedContentBackgroundColor))` to the sidebar `List` to recolor selection fill.

**The plan compiled cleanly but produced no visible change.** Investigation revealed `.tint(_:)` doesn't propagate to `NSTableView`'s source-list selection rendering on macOS 26 Tahoe — sidebar selection is painted by AppKit underneath SwiftUI, using `controlAccentColor` from the asset catalog directly; SwiftUI's tint environment value is bypassed. The empirical bet at the heart of the plan failed.

Compounding the perception: a stale Pommora process from the prior session was Xcode-debugger-held, surviving SIGKILL. Nathan was looking at the OLD instance while the new build ran invisibly behind it. Fix for the stuck process: kill its parent `debugserver` (LLDB) — the orphan is then reaped by init.

**Pivot:** reverted to pre-plan state; replanned with the minimum-viable scope (just the `.listRowInsets` deletion, which was the one change that did work). Selection-color customization deferred to a future iteration with a documented post-mortem in Sidebar.md.

During re-execution, Nathan flagged that the chevron rotation on the Collections `DisclosureGroup` wasn't animating. Cause: `collectionExpansion[c]` was bound via a custom `Binding` with a `set:` closure, and custom Bindings don't auto-animate. Fix: wrap the dictionary write in `withAnimation(.smooth(duration: 0.25))`.

**Brief excursion into Finder pattern.** With `.listRowInsets` removed, flat rows still showed the chevron-column gap that's reserved by the presence of `DisclosureGroup` in a `.listStyle(.sidebar)` List (Mail / Xcode pattern). Nathan thought Items + Spaces shouldn't reserve chevron space. A spike dropped `DisclosureGroup` and hand-rolled Collection expansion with an inline chevron `Button` + conditional member `ForEach` — produced the Finder flush-left pattern. Nathan liked the look but then realized Apple's default DOES have the slight indent and reverted. Captured the spike as documented future-direction in [Sidebar.md](../ Features/Sidebar.md): tighter chevron-to-icon spacing than Apple's default, with the rest of the sidebar matching the tighter visual.

**Final polish:** Nathan noticed section header text reading darker than Mail's "Favorites" reference. Tried removing `.scrollContentBackground(.hidden)` first (no clear improvement; restored). The fix: switch `Section("Title", isExpanded:)` shorthand to the custom-header form with explicit `Text("Title").foregroundStyle(.secondary)`. The shorthand inherits a darker default header style on the `.hidden`-backdrop surface; explicit secondary brings it to Mail-reference tone.

Also added Spaces section collapsibility (`isExpanded: $spacesExpanded`) to match Collections.

#### Key technical findings

- **`.tint(_:)` does not propagate to `NSTableView` source-list selection on macOS 26 Tahoe.** Sidebar selection fill is painted by AppKit directly from `controlAccentColor`. Recoloring requires either AppKit introspection (find underlying `NSTableView`, override `selectionHighlightStyle` + force `isEmphasized = false` via a row-view subclass) or a custom selection layer built from `@State` + `.listRowBackground` after dropping `List(selection:)`. Both were judged out of scope for v0.0 chrome polish.

- **`DisclosureGroup` presence in a `.listStyle(.sidebar)` List triggers chevron-column reservation across every row in that List.** This is the Mail / Xcode pattern (icons align horizontally; flat rows have empty chevron-column space). Finder pattern (flat rows flush-left, only "folder" rows show chevron + indent) requires dropping `DisclosureGroup` entirely.

- **Custom `Binding` instances don't auto-animate.** When `isExpanded` is bound to a `[Int: Bool]` subscript via a hand-rolled Binding's `get`/`set` closures, the toggle runs outside SwiftUI's animation transaction. Wrap the write in `withAnimation(_:)` explicitly. Direct `@State` `$binding` doesn't need this.

- **`Section("Title", isExpanded:)` shorthand uses a darker default header style** than the explicit `Section(isExpanded:) { } header: { Text("Title").foregroundStyle(.secondary) }` form on a `.scrollContentBackground(.hidden)` surface in macOS 26 Tahoe. For Mail-reference tone, use the custom-header form.

- **Xcode-debugger-held processes refuse SIGKILL.** STAT field shows `X`; parent is a `debugserver` (LLDB). Kill the parent to release the orphan. This was the cause of stale-build confusion mid-session and now lives as the "single-instance enforcement" sequence in any builder-agent dispatch (kill by executable path → check STAT → escalate to debugserver-parent kill).

- **HStack spacing ≠ row indent.** `HStack(spacing:)` controls the gap between sibling views inside the HStack. Row indentation comes from `.padding(.leading, N)` or `.listRowInsets(EdgeInsets(...))`. Chevron-column reservation comes from `DisclosureGroup` presence in the same sidebar List. Each adjustment has its own mechanism; they're not interchangeable. Captured as working vocabulary in Sidebar.md.
