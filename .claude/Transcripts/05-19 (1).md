#### New architectural rule (CRUD-Patterns.md)

**Preview-window prerequisite:** PreviewWindow primitive must ship for an entity kind BEFORE any "open in preview" UI is wired for that kind. CRUD on an entity may land independently. The v0.2.7.2 EntityWindowHost is the cautionary tale — feature-specific window plumbing that rotted as soon as requirements shifted.

#### Bugs caught + fixed live

1. **Double-click didn't open anything** — root cause was `.onTapGesture(count: 2)` getting intercepted by SwiftUI List's underlying NSTableView selection handler on macOS. Fix: `.simultaneousGesture(TapGesture(count: 2))`.
2. **Collections specifically didn't open** — leftover `case .collection: return nil` in `SidebarSelection.init?(stateRef:)` from the v0.2.7.2 "collections not wired" decision. Wired it.
3. **Routing through `AppGlobals.mainWindowRouter` @Observable from the popover view host wasn't propagating reliably** — same root cause as the empty-Recents bug fixed by the snapshot pattern. Bypassed by passing `onOpen: (SidebarSelection) -> Void` closure from ContentView; NavDropdownButton calls the closure directly, which writes to ContentView's @State via the captured setter. MainWindowRouter stays for back/forward (separate code path that works fine).
