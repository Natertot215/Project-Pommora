## What happened along the way

Started subagent-driven per the plan, but Phase 2 (vendoring the engine into Pommora's source tree) ran into cascading Swift 6 strict-concurrency errors — the engine targets Swift 5.9, Pommora is Swift 6 + ExistentialAny. Spent a chunk of agent dispatches diagnosing `NSTextLayoutFragment` override isolation mismatches before Nathan stepped in:

> "All agents must be 4.7; if doing this in one context window is better then you can do it without agents if it produces higher quality results"

Pivoted strategy: moved the engine to `External/MarkdownEngine/` as a **local Swift Package** (Swift 5.9 mode), avoiding the cascade entirely. Five minimal `@MainActor` annotations were enough — engine builds standalone clean. That unblocked Phases 1, 2, 4, 4.5 cleanly (`h.1` → `h.4`). Phase 3 substantive (full AST tokenizer/styler rewrite) was deferred to v0.2.7.1.

After ship + first launch, Nathan went into rapid live-feedback mode. Each round caught a real polish issue:

> "table still renders as markdown"  
> "I shouldnt be able to do this haha; it should just auto transform it into a seperator on ---, not just add more dashes like it currently does"  
> "we want all available formats to be selectable via the right-click inspect"

`h.7` shipped UX fixes (title-body padding, body 24pt textInsets, auto-unpair on backspace). `h.8` added the Apple-AST supplemental styler (BlockQuote/Strikethrough/Table/ThematicBreak rendering composed on top of the regex tokenizer) and expanded the right-click menu with Format / Heading H1–H6 / Lists / Block submenus. `h.9` made `---` render as a real horizontal line via a custom `MarkdownTextLayoutFragment.draw` hook + hid table pipes/separator row + wired Enter-on-title → body focus shift.

`h.10` chased two stubborn bugs:
- HR "deletes itself" → root cause was `drawThematicBreak` only checking the first char of the fragment range; switched to `enumerateAttribute` scan.
- Title-Enter selected-all instead of moving focus → added `@FocusState` + explicit `titleFocused = false` before the makeFirstResponder walk.

Plus Nathan caught H5/H6 rendering smaller than body text:

> "lets remove H 6; it shows up smaller than body text" / "and H5"

Trimmed heading submenu to H1–H4.

After the dust settled, Nathan's tone shifted:

> "Im honestly just super stoked about how well this looks fo rnow, its much better than I though!"

Session close: tagged `v0.2.7.0`, pushed `main` + tag to origin (first push since v0.2.0 series), wrote `Features/PageEditor.md` as the editor implementation spec, and bundled my doc updates with Nathan's parallel-session NavDropdown work (the v0.2.8 → NavDropdown pivot — `NavDropdown.md` new, `Navigation-Bar.md` deleted) into the `h.11` ship-out commit. 11 commits total this session.
