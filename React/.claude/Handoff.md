### Lessons learned

- **One repo now — no more subtree sync.** React used to live in a separate `Pommora - React` checkout subtree-merged into `React/`; that split let edits land on one side invisibly to the other. It's **consolidated**: `React/` in the monorepo is the single source — no mirror to keep in sync.
- **vanilla-extract vars can't be read from plain CSS** — `createGlobalTheme` emits hashed names. Bridge via `globalStyle(':root', { vars: { '--name': token } })` (`theme-vars.css.ts`) to expose stable `var(--…)` — one source across `.ts` and `.css`.
- **Figma's 10-mode cap is hard** (`addMode` → "Limited to 10 modes only"). To add an 11th chip color (red), hand-build a gallery column with raw paints, not a mode. The Color collection's `defaultModeId` is also read-only.
- **Verify usages — fills AND strokes — before deleting a Figma variable.** A fills-only sweep left 12 stroke bindings on `label-on-color`, so deleting it left dangling refs. A claim is a hypothesis until checked.
- **Tints are tokenized** (`tint.ts`): an opacity scale — `tint-primary 60 / secondary 40 / tertiary 20 / quaternary 15 / solid 100` — applied to any base via `tintAt(base, step)`. The `chip` recipe maps onto it: fill = primary · stroke = secondary (2px; **1.5px** Checkbox) · text = `label-primary` washed with quaternary of the base (label with tint, not white with tint). `tint-tertiary` → tinted segments, `tint-solid` → tinted buttons (reserved). The base `chip` **composes `text.control.emphasized`** (never re-states the ramp).
- **Gallery instances can carry local text-style overrides that survive a component edit** — verify Figma galleries at the *instance* level, not just the component; bind real type to a live style (never `NONE/MIXED`). SF Pro icon glyphs are size-only — never bind an Inter style or they tofu.
- **Keep design surfaces registry-shaped.** The showcase iterates the `vars.color.*` groups (via `COLOR_GROUPS`), `text`, `chipColor`, the `icons` map, and materials, so a new token / icon appears with no showcase edit.
- **Derive the accent, don't restate it.** `--accent-fill` / `--accent-text` are `color-mix(var(--accent) …)`, so the runtime sets ONE property (`--accent`) and fill/text cascade. `color-mix(in srgb, var(--accent) 70%, white)` reproduces lavender's Figma `accent-text` and generalizes to any accent.
- **System accent is per-runtime.** Electron `systemPreferences.getAccentColor()` returns `RRGGBBAA` (RGB = first 6 chars; macOS has no live change event → read at load); the web showcase reads the CSS `AccentColor` system color off a probe element.
- **JS/Electron config stays at the repo root** (`package.json`, lockfile, `tsconfig*`, `vite.config.ts`, `electron.vite.config.ts`, `node_modules`) — hard-expected by the toolchain. `out/` · `.vite` · `*.tsbuildinfo` · `.DS_Store` are gitignored cruft.
- (Data layer) Keep greenfield multi-agent stages sequential + self-verified green; `ELECTRON_RUN_AS_NODE=1` breaks GUI launches (strip it); CommonJS main/preload.
- **Packaged Electron renderer must NOT use `file://`.** Vite emits ES-module scripts (`type=module` + `crossorigin`); over `file://` every resource is an opaque origin, so module CORS blocks them → blank window ("a chrome instance"). Serve the bundle over a registered secure **`app://` scheme** (`fs.readFile`, asar-aware) in `main/index.ts`; dev (`ELECTRON_RENDERER_URL` http) is unchanged. Diagnose a suspected blank build by serving `out/renderer` over http — if the bundle renders, it's a `file://`-delivery bug, not a code bug.
- **macOS Tahoe masks every app icon into its own rounded tile** — so icon art must be **full-bleed** (dark edge-to-edge, no self-rounded squircle/bevel) or it double-borders. `scripts/make-icon.mjs` crops a beveled-squircle source to full-bleed; `OVER` = zoom/crop.
- **`open` DOES launch the packaged app in Nathan's GUI session** — `env -u ELECTRON_RUN_AS_NODE open release/mac-arm64/Pommora.app` hands off to LaunchServices and the window appears for Nathan (verified this session). Relaunch a fresh build by `pkill -f "release/mac-arm64/Pommora.app/Contents/MacOS/Pommora"` first (the React one — NOT the Swift Pommora under Xcode/DerivedData). **Direct-exec** (`./node_modules/.bin/electron .`) still won't render from the agent shell (no Aqua session). So the agent can repackage + relaunch for Nathan, but can't SEE the result — visual verification is his.
- **better-sqlite3 is dual-ABI: Electron ≠ Node, even at the same Node major.** One `node_modules` holds one ABI; vitest runs under system Node, the app under Electron. So `node_modules` stays **Node-ABI by default** (the test gate is reliable) and electron-builder rebuilds for Electron at package time. Verify a native load headlessly with `ELECTRON_RUN_AS_NODE=1 ./node_modules/.bin/electron -e "new (require('better-sqlite3'))(':memory:')"`. `db.ts` **lazy-requires** better-sqlite3 inside `openDb`'s try/catch, so an ABI mismatch (or a packaged binary built for the wrong runtime) degrades to a null db (cold index) instead of throwing at module-eval and crashing the app — the index is off the read path, so the write path works regardless. (electron-builder's `buildFromSource=false` rebuild proved inconsistent → the packaged index may be cold; fine until Views consume it.)
- **Live index = full-refresh, not incremental.** After each mutation, `refreshSessionIndex` drops `index.db` + cold-rebuilds via `rebuildIndex` — correct by construction, reusing `buildIndex` with zero per-entity row-logic duplication. Incremental upserts/deletes were rejected for v1 (they'd duplicate `buildIndex`'s orphan/FK/connection knowledge); the once-built `deleteEntity` primitive was removed as unwired. Incremental is a v1.1 perf optimization for when nexuses grow + a query consumer exists.
- **The watcher watches `.nexus/`, ignores only `index.db*`.** The live filesystem watcher (Phase 4) must watch `.nexus/` — Contexts (`.nexus/<tier>/`) + settings/accent/labels/order live there — so blanket-ignoring `.nexus` (to dodge the WAL-churning `index.db`) silently kills external-change refresh for those. Ignore `index.db*` + `.trash` + dotfile cruft specifically. And a chokidar watcher MUST have an `.on('error')` or an `EMFILE`/`ENOSPC` (fd/inotify exhaustion) re-throws as an unhandled EventEmitter error → crashes the whole main process.

### Fix log

- **PommoraDND drop "jerk on set"** — items lurched at commit. Two causes, both fixed: (1) transitions stayed on at rest, so the transform-reset animated while the reorder moved the DOM → snap-then-reanimate; gate transitions to a live drag only (idle = instant snap). (2) The commit ran on a blind `setTimeout(duration)`, but the CSS transition starts a frame later, so the timer fired while items were still mid-flight and snapped them short (worse after aggressive dragging) → commit on the lifted item's `transitionend` (fallback timer for the no-transition case). Residual sub-pixel micro-jerk accepted. Same `transitionend` pattern used for the cross-list overlay's fly-to-slot.
- **Glass lens default position** — measured the forest tile before the swatch/type rows grew (their labels populate after mount), so it landed ~163px high; fixed by re-measuring via a `ResizeObserver` until the first drag.
- `label-on-color` removal: first sweep was fills-only → 12 stroke bindings missed + the variable left dangling. Re-swept fills + strokes (12 vectors → `label-primary`), then deleted.
- Per-color cleanup: usage-check showed **all 55 in use** (lavender = accent, 100+ uses) → migrated (accent → semantic; gallery → modes; spectrum → solids) before deleting **54** tint variables.
- (Data layer) `SchemaTransaction` rollback hole + `.bak-` sweep (`d712999`); index `target_kind` = `area`/`topic`/`project`, not `context_tier` (`8309e20`).
- **Blank packaged app ("a chrome instance")** — renderer loaded over `file://` → ES-module CORS block. Fixed with the `app://` scheme (`ae31cbf`).
