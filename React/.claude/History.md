### Callouts â€” `> [!callout]` boxes with nested syntax (2026-06-27)

A callout is a `> [!type]` blockquote (GitHub/Obsidian convention, single `[!callout]` type for now) rendered as a **bordered, gutter-width box**, typed with the `||` shorthand. Chosen over a single-cell-table scheme (Option B): the blockquote form degrades to a readable quote in any markdown app and stays agent-legible, while a `â€–`/`==` box would show literal border chars + unrendered inner syntax. â†’ `Features/MarkdownPM.md` آ§ Callout.

**Nested syntax renders inside the box** via a chrome/construct split: each line's `>` prefix is stripped and the inner content renders as its own construct (a `> - x` is a real bullet, `> # h` a heading, `> ---` an inner rule spanning the box's inner gutter), indent measured from the box edge through one shared `--li-origin`. A leading bullet/HR widget **absorbs the prefix into one replace** â€” CM drops a widget-replace that merely *touches* a preceding replace, which is why only `-` bullets had failed to render. Every input op (continuation, Tab-indent, smart-backspace, checkbox-canonicalize, drag-reorder, the dash auto-formats) was made **prefix-aware** so a list behind a `>` behaves like a top-level one.

**Detection is per-HEAD, not per-run** (the robustness lesson): any `[!type]` line starts its own callout, so adjacent / hand-typed / pasted / quote-nested heads never merge into one box leaking a raw `[callout]` tag. The hidden `> [!type] ` head is an **atomic range** â€” the caret can't enter it, so typing / forward-delete / backspace can't corrupt the discriminator and silently demote the box to a quote.

**Hardened by an exhaustive live-fuzz pass** (every fix screenshot-verified over CDP): `||` no longer eats trailing line content or merges with an adjacent box; Shift+Enter stays boxed only when the whole selection is inside; the en-dash auto-format no longer eats a `- ` bullet behind a `>`; drag re-leads correctly across box boundaries (`boxDepth` + `LEAD_RE`) and the drop-line stops at the box's inner edge; the drawn caret clamps its height next to a block widget. Multiple audit agents + a code review + a simplify pass.

**Chrome polish + code blocks (same day):** fenced **code blocks render inside callouts** (the fence scanner pairs by quote-depth â€” a `> ``` is closed only by a `> ``` â€” so a top-level block quoting a fence can't corrupt; the code keeps its fill as an inset layer inside the box gutter). A **blockquote nested in a callout** (`> > â€¦`) renders as its own inset quote block (bar + tint via `::after`) instead of flattening to body text. The box border moved to an inset **`::before`** so two stacked callouts get a `--callout-gap` of separation (like lists/headings) without a caret-breaking margin; a first-line callout drops that top gap (the title-bar header provides it). One **`--li-origin`** var is the box's inner gutter for every inner construct (DRY). A **hover grip** sits in the top-left gutter (table-style fade-in; click actions deferred). **Deferred (finicky, tracked in `Features/MarkdownPM.md`):** a **table** inside a callout still renders raw â€” the table widget's region detection isn't prefix-aware; the box stays intact around it.

### List drag-to-reorder by the glyph (2026-06-26)

Bullet / ordered / checkbox list items reorder by **grabbing their glyph** and dragging â€” the sidebar's PommoraDND interaction (grab cursor, in-place dim, an accent insertion line that follows the target's indent out to the right gutter), but driven against CodeMirror line geometry and committed as a **source-line move**, not a DOM reflow ("the DND animation just goes over the real file"). **The load-bearing gotcha (locked):** CodeMirror's `WidgetType.ignoreEvent()` **defaults to true** â€” a widget swallows every DOM event from its own DOM â€” so the bullet/checkbox glyphs (replace-widgets) need an explicit `ignoreEvent â†’ false` or pointerdown never reaches the drag handler (the bug that made bullet-drag silently dead while ordered numbers, which are styled source text not a widget, worked).
